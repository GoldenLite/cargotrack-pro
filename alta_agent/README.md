# Alta-агент для CargoTrack Pro

Маленький Python-скрипт, который опрашивает API CargoTrack Pro,
скачивает XML-документы из очереди и кладёт их в hot-folder Альта-ГТД.

```
CargoTrack Pro ──HTTPS── alta_agent.py ──FS──> C:\ALTA\inbox\
   (VPS)                  (на ПК с Альтой)
```

С точки зрения корпоративной сети это **исходящий HTTPS-трафик** — то же
самое, что вы ходите в веб-сервис через браузер. Никаких открытых портов,
никаких VPN.

## Установка (Windows)

1. Скопируй папку `alta_agent` на ПК, где установлена Альта-ГТД
   (например, в `C:\AltaAgent`).

2. Установи Python 3.10+ если ещё нет: https://www.python.org/downloads/
   При установке поставь галку "Add to PATH".

3. Скопируй `alta_agent.ini.example` → `alta_agent.ini`, открой блокнотом,
   заполни:
   - `base_url` — адрес твоего CargoTrack Pro (например, `https://cargotrack.your-domain.ru`)
   - `token` — длинный случайный токен (см. ниже)
   - `hotfolder` — папка, которую Альта мониторит (Альта → Настройка →
     Внешние программы → Загрузка → Путь к папке «Автозагрузка»)

4. На стороне Django-сервиса добавь в `.env` ту же строку:

   ```
   ALTA_AGENT_TOKEN=твой_токен
   ```

   Сгенерировать токен можно так:

   ```
   python -c "import secrets; print(secrets.token_urlsafe(48))"
   ```

5. Запусти агент вручную для проверки:

   ```
   python alta_agent.py
   ```

   В логе должно появиться:
   ```
   Старт. base_url=... hotfolder=... interval=4s
   ```

   Нажми в CargoTrack Pro «Отправить в Альту» на любой накладной — через
   несколько секунд агент должен залогировать `#NN ...: SENT`, а файл —
   появиться в hot-folder. Альта подхватит его сама.

## Автозапуск (Планировщик задач Windows)

Чтобы агент работал в фоне без открытого окна:

1. Открой "Планировщик заданий" → Создать задачу
2. Общие → название "CargoTrack Alta Agent". Поставь:
   - Выполнять только для вошедшего пользователя
   - Запускать с наивысшими правами (не обязательно)
3. Триггеры → Создать → "При входе в систему" → ОК
4. Действия → Создать →
   - Программа: `python.exe` (или полный путь — `C:\Python313\python.exe`)
   - Аргументы: `C:\AltaAgent\alta_agent.py`
   - Рабочая папка: `C:\AltaAgent`
5. Условия → сними галку "Запускать только при питании от сети", если ноут
6. Параметры → "Если задача уже выполняется → Не запускать новую копию"
7. ОК. Перезагрузи / выйди-войди — агент запустится сам в фоне.

Логи смотри в `alta_agent.log` рядом со скриптом.

## Безопасность

- Токен хранится только в `alta_agent.ini` и в `.env` Django. Если он утёк —
  смени его в обоих местах. Старый токен сразу станет недействителен.
- Агент **только забирает** документы — никакой обратной связи с ПК на
  сервер, кроме `ack`/`fail` по тем же ID. Скомпрометированный токен не
  даёт доступ к ПК.
- Hot-folder лучше держать на локальном диске, не в OneDrive — Альта может
  пытаться прочитать файл, пока OneDrive ещё не закончил синхронизацию.

## Отладка

- `alta_agent.log` рядом со скриптом — все события.
- Страница `/alta/queue/` в CargoTrack Pro — что в очереди, что отправлено,
  что упало (с кнопкой "Вернуть в очередь" для ретрая).
- Если запись долго в статусе "В очереди" — почти всегда агент не запущен
  или не видит сервер. Проверь, что `python alta_agent.py` запускается
  вручную без ошибок.

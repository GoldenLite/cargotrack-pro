Что такое бизнес-процесс?

Бизнес-процесс (воркфлоу) — это заранее описанный сценарий движения груза или накладной по этапам. Каждый процесс состоит из шагов (этапов обработки), переходов между ними и автоматизаций (действий, которые выполняются при наступлении события).

Пример: Воркфлоу «B2B Импорт»

Процесс автоматически запускается для каждой накладной B2B-импорта и продвигается вперёд при смене статуса накладной.
Автоматический запуск

Когда создаётся новая партия или накладная, система автоматически проверяет все активные процессы и запускает те, которые подходят по условиям.

Настройка Описание Пример
Тип сущности К чему применяется процесс Партия (MAWB) или Накладная (HAWB)
Условия запуска Поля сущности, которым должна соответствовать запись cargo_type = B2B
shipment_type = IMPORT
Автозапуск Запускать автоматически при создании записи Включено / Выключено
Если «Автозапуск» выключен — процесс можно запустить вручную со страницы накладной или партии через кнопку «Запустить процесс».
Шаги и продвижение по процессу

Каждый шаг привязан к конкретному статусу сущности. Когда статус меняется — процесс автоматически переходит на соответствующий шаг.

Тип процесса Поле шага Пример значения
Для партии (Cargo) Этап партии DRAFT → FORMED → DISPATCHED → ARRIVED → CUSTOMS → RELEASED
Для накладной (HAWB) Логистический статус AT_SVH, IN_TRANSIT, IMPORT_CUSTOMS, RELEASED…
Таможенный статус PENDING, DECLARED, INSPECTION, RELEASED…
Шаг без входящих переходов считается начальным, шаг без исходящих — финальным. Когда процесс достигает финального шага, экземпляр автоматически закрывается со статусом «Завершён».
Переходы между шагами

Переход определяет, при каком условии можно перейти от одного шага к другому. Переходы бывают двух видов:

1
Автоматический переход — срабатывает при смене статуса сущности. Ничего делать не нужно — система сама сдвигает экземпляр процесса вперёд.
2
Ручной переход — пользователь нажимает кнопку действия на экране (пока в разработке).
Автоматизации

К каждому переходу можно привязать одно или несколько действий, которые выполняются автоматически при наступлении перехода.

Тип действия Что делает
send_notify Отправляет уведомление — email или внутреннее сообщение
change_stage Автоматически меняет этап партии на указанный
set_field Устанавливает произвольное поле сущности в нужное значение
assign_user Назначает ответственного пользователя на запись
webhook Вызывает внешний URL с данными о событии (HTTP POST)
Как создать процесс: пошагово
1
Нажмите «Создать процесс», введите название и описание.
2
Откройте процесс в редакторе. В панели «Настройки процесса» (слева) выберите: тип сущности (Партия или Накладная), условия автозапуска (если нужны) и включите «Автозапуск».
3
Добавьте шаги через кнопку «+ Добавить шаг». Для каждого шага укажите имя и соответствующий статус (этап партии или статус накладной).
4
Соедините шаги переходами: потяните от одного блока к другому в редакторе схемы.
5
Добавьте автоматизации к переходам при необходимости.
6
Нажмите кнопку активации процесса — после этого он начнёт применяться к новым записям.
Прогресс выполнения процесса виден на странице каждой партии и накладной в блоке «Бизнес-процессы». Там же можно запустить процесс вручную или отменить активный экземпляр.
